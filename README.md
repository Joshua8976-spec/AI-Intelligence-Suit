AI-Powered Intelligence Suite: Finance & NLP Evolution
A Full-Stack journey from Command-Line Logic to Deep Learning Dashboards.

🚀 Project Overview
This repository contains two cornerstone projects developed to explore the intersection of Machine Learning, Quantitative Finance, and Natural Language Processing. These projects document an evolutionary engineering process, starting with core algorithmic scripts in the terminal and scaling into production-ready Flask web applications.

📈 Project 1: Financial Intelligence Hub (Deep Learning)
A comprehensive wealth-management dashboard that merges live market data with a Recurrent Neural Network (RNN).

Key Features
Predictive AI: Uses an LSTM (Long Short-Term Memory) network built in PyTorch to forecast stock trends.

Real-Time Tracking: Integration with the yfinance API for live asset valuation.

Data Engineering: Custom VWAP (Volume-Weighted Average Price) algorithm to consolidate transaction history.

Visualization: Interactive asset allocation charts powered by Plotly.

📽️ Project 2: Movie Discovery & Sentiment Engine (NLP)
A semantic search engine that understands the "essence" of cinema through text analysis and weighted quality metrics.

Key Features
Vector Space Modeling: Utilizes TF-IDF Vectorization and Cosine Similarity to map thematic links between 5,000+ films.

Quality Filtering: Implemented an IMDb-based Weighted Rating Formula to prioritize high-quality recommendations.

Sentiment Analysis: Integrated VADER Sentiment module to quantify audience reception and provide "Mood Scores."

🛠️ Tech Stack
Languages: Python 3.14

AI/ML: PyTorch, Scikit-Learn, NLTK

Data: Pandas, NumPy, yfinance API

Web: Flask, Jinja2, HTML5, CSS3, Plotly

🧠 Engineering Reflections (The "Ivy League" Edge)
Architectural Resilience
During the development of the Finance Hub, I encountered significant OS-level DLL conflicts with TensorFlow on Python 3.14. I successfully navigated this by migrating the entire backend to PyTorch, demonstrating the ability to adapt to framework limitations and resolve low-level system errors.

Iterative Growth
Both projects followed a strict development lifecycle:

Phase 1 (CLI): Logic verification in the Command Line Interface.

Phase 2 (Data): Implementation of persistent storage (CSV/TXT) and API bridges.

Phase 3 (UI): Full-stack integration into a dynamic web environment.

🔮 Limitations & Future Research
Black Swan Events: Currently, the LSTM model relies on historical trends; future work includes integrating News Sentiment Analysis to account for unpredictable global events.

Semantic Depth: Planning to migrate from TF-IDF to Transformer-based models (BERT) for deeper narrative understanding in the Movie Engine.
